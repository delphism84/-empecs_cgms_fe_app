EMPECS 소셜 로그인 설정 (복사·붙여넣기 전용)

이 문서에 적힌 글자만 각 사이트 입력칸에 붙여 넣으세요. 주소·도메인·앱 이름을 스스로 정하거나 바꾸지 마세요.

꼭 지켜 주세요
• 빈 칸이 있는데 이 문서에 값이 없으면 임의로 만들지 말고 EMPECS 개발 담당자에게 문자·메일로 요청하세요.
• SHA-1 인증서 지문, Android 키 해시는 개발 쪽에서만 만들 수 있는 값입니다. 담당자가 보내 준 한 줄을 그대로 붙여 넣습니다.

① 공통 — 무조건 이 값만 사용

아래 각 줄 옆 복사를 누른 뒤, 콘솔 화면의 같은 이름의 칸에 붙여 넣습니다.

서비스 주소 · 홈페이지 URL · OAuth 동의화면에 묻는 "앱 홈페이지" 등 (https 포함) https://empecs.lunarsystem.co.kr
Google — 승인된 자바스크립트 출처 (한 줄) https://empecs.lunarsystem.co.kr
Google — 승인된 리디렉션 URI (한 줄) https://empecs.lunarsystem.co.kr/api/oauth/google/callback
Apple — Domains and Subdomains (https 없이 도메인만) empecs.lunarsystem.co.kr
Apple — Return URLs (한 줄, https 포함) https://empecs.lunarsystem.co.kr/api/oauth/apple/callback
Kakao — Redirect URI (한 줄) https://empecs.lunarsystem.co.kr/api/oauth/kakao/callback
Android 앱 — 패키지 이름 (Google Android 클라이언트 / Kakao 플랫폼 Android) com.helpcare.app
iOS 앱 — 번들 ID (Apple App ID / Kakao 플랫폼 iOS) com.helpcare.helpcare

GOOGLE Google Cloud — 할 일 순서

사이트: console.cloud.google.com

  1. 구글 계정으로 로그인합니다.
  2. 상단에서 프로젝트를 선택하거나 새로 만듭니다.
  3. 왼쪽 메뉴 API 및 서비스 → 사용자 인증 정보로 이동합니다.
  4. 처음이면 화면 안내에 따라 OAuth 동의 화면을 완료합니다. 홈페이지·개인정보처리방침 URL을 묻는 칸이 있으면 위 ① 공통의 https://empecs.lunarsystem.co.kr만 사용합니다.

A. 웹 애플리케이션 클라이언트 만들기

  1. + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID → 애플리케이션 유형 웹 애플리케이션을 선택합니다.
  2. 승인된 자바스크립트 출처에 ①에서 복사한 https://empecs.lunarsystem.co.kr 한 줄을 붙여 넣습니다.
  3. 승인된 리디렉션 URI에 ①에서 복사한 …/api/oauth/google/callback 한 줄을 붙여 넣습니다.
  4. 저장 후 나오는 클라이언트 ID·필요 시 클라이언트 보안 비밀을 EMPECS 담당자에게 전달합니다. (이 두 줄은 복사 버튼이 없습니다 — 화면에서 복사하세요.)

B. Android 클라이언트 만들기 (모바일 앱용)

  1. 다시 + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID → 유형 Android를 선택합니다.
  2. 패키지 이름에는 ①의 com.helpcare.app만 붙여 넣습니다.
  3. SHA-1 인증서 지문 칸에는 EMPECS 담당자가 보내 준 값 한 줄만 붙여 넣습니다. 담당자에게 받기 전에는 이 단계를 보류합니다.
  4. 저장 후 나오는 클라이언트 ID를 EMPECS 담당자에게 전달합니다.

APPLE Apple Developer — 할 일 순서

사이트: developer.apple.com (유료 멤버십 필요)

  1. Certificates, Identifiers & Profiles → Identifiers로 갑니다.
  2. App IDs에서 EMPECS iOS 앱용 항목을 선택하거나 새로 만듭니다. Bundle ID는 반드시 ①의 com.helpcare.helpcare와 동일하게 맞춥니다.
  3. 기능에서 Sign In with Apple을 켜고 저장합니다.
  4. 웹·Android에서 Apple 로그인까지 쓰는 경우: Identifiers에서 Services IDs를 추가하고, Sign In with Apple을 설정합니다.
  5. Services ID 설정 화면에서 Domains and Subdomains에 ①에서 복사한 empecs.lunarsystem.co.kr만 넣습니다.
  6. Return URLs에 ①에서 복사한 https://empecs.lunarsystem.co.kr/api/oauth/apple/callback 한 줄만 넣습니다.

맥·Xcode에서 앱 서명·빌드는 EMPECS 개발 쪽에서 진행합니다. 고객은 위 웹 콘솔에 표시된 ID·키만 발급·전달해 주시면 됩니다.

KAKAO Kakao Developers — 할 일 순서

사이트: developers.kakao.com

  1. 내 애플리케이션에서 앱을 만들거나 선택합니다.
  2. 앱 설정 → 플랫폼에서 Android를 추가합니다. 패키지명에 ①의 com.helpcare.app만 넣습니다.
  3. 마켓 URL 등 이 문서에 없는 칸이 필수(빨간 표시)로 나오면, 임의로 적지 말고 EMPECS 담당자에게 값을 요청합니다.
  4. Android 키 해시 칸에는 EMPECS 담당자가 보내 준 값만 붙여 넣습니다. 여러 줄이면 담당자 안내 순서대로 모두 등록합니다.
  5. iOS 플랫폼을 추가하고 번들 ID에 ①의 com.helpcare.helpcare만 넣습니다.
  6. 제품 설정 → 카카오 로그인을 ON으로 켭니다.
  7. Redirect URI에 ①에서 복사한 https://empecs.lunarsystem.co.kr/api/oauth/kakao/callback 한 줄을 추가·저장합니다.
  8. 앱 키 화면의 네이티브 앱 키(및 필요 시 REST API 키)를 EMPECS 담당자에게 전달합니다.
담당자에게 넘길 때
Google / Apple / Kakao 화면에서 발급된 클라이언트 ID, 키, Services ID 등은 스크린샷이나 메모로 임의 요약하지 말고, 화면에 보이는 문자열을 그대로 복사해 전달해 주세요.
